# Security Policy

## Supported Versions

Security fixes are currently provided for the latest published version only.

## Reporting a Vulnerability

Please report vulnerabilities privately and do not open a public issue.

1. Email: **alexandramcgarryx@gmail.com**
2. Include:
- A clear description of the issue.
- Reproduction steps or proof of concept.
- Potential impact.

You can expect an acknowledgement within 5 business days.

## Disclosure Process

- Valid reports will be triaged and fixed as quickly as possible.
- A patched version will be published to npm.
- Public disclosure should happen after a fix is available.
